Порядок действий для нахождения ошибок:

Запустил отладчик:
jdb Main
 
 
Устанавил точки останавки в методах:
stop in model.BankAccount.deposit
stop in model.BankAccount.withdraw
stop in model.BankAccount.transfer
stop in model.BankAccount.applyInterest


Запустил программу:
run


При остановке на каждой точке проверял значения переменных:
print balance
print amount


Логически сравнивал полученные значения с ожидаемыми:
При внесении средств (deposit) баланс должен увеличиваться на указанную сумму.
При снятии (withdraw) — уменьшатся, но не быть отрицательным.
При периводе (transfer) — сумма должна сниматся со счёта отправителя и добавляться получателю.


Где находились ошибки:
withdraw(double amount) — метод снятия денег со счёта.
transfer(BankAccount target, double amount) — метод перевода денег между счетами.
applyInterest(double rate) — метод начисления процентов на баланс.


Описание ошибок:

1.Метод withdraw() 
неправильная проверка условия при снятии средств. Была проверка, что баланс меньше нуля, хотя нужно проверять, хватает ли денег для снятия.(можно снять больше, чем есть на счёте)
Добавил проверку:
if (amount > balance) throw new InsufficientFundsException()

2.Метод transfer() 
Порядок действий неправильнй: сначала выполнялось зачисление денег на другой счёт, а уже потом списание с текущего.(нарушает логику)
Добавил строку:
this.balance -= amount; перед вызовом target.deposit(amount)

3.метод applyInterest()
При начислении % на балик вызывался deposit(balance + interest), который увеличивал сумму неверно.(к балику добавлялся не только %, но и сам балик ещё раз.)
