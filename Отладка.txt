Порядок действий для нахождения ошибок:

Запустил отладчик:
gdb ./student_manager_debug


Установил точки останова в ключевых функциях:
break add_grade
break find_student
break print_student


Запустил программу:
run


При остановке на каждой точке проверял значения переменных:
print manager->count
print i
print student->grades_count
print student->average


Логически сравнивал полученные значения с ожидаемыми:
Количество студентов не должно выходить за пределы массива.
Добавление оценки должно корректно выделять память и записывать значение.
Средний балл должен считаться как сумма оценок, делённая на их количество.
Функция поиска студента должна проверять только существующих студентов.
Вывод оценок должен проходить по правильному количеству элементов.


Где находились ошибки:
add_grade(StudentManager* manager, int student_id, float grade) — функция добавления оценки студенту.
find_student(StudentManager* manager, int student_id) — функция поиска студента по ID.
print_student(StudentManager* manager, int student_id) — функция вывода информации о студенте.


Описание ошибок:
1. цикл в add_grade
Цикл for (int i = 0; i <= manager->count; i++) в функции add_grade идёт до manager->count включительно.
Это приводит к выходу за грыницы массива студентов, так как индексы массивов начинаются с 0 и 
заканчиваются count - 1. При обращении к manager->students[i] на ласт итерации возможен доступ 
к неинеициализированной памяти.
2. realloc для добавления оценки
Вызывается student->grades = (float*)realloc(student->grades, student->grades_count * sizeof(float));.
Здесь выделяется памяти ровно столько, сколько уже есть оценок, без места для новой. При записи новой
оценки может праизойти выход за пределы выделенной памяти.
3. вычисление среднего балла
Строка student->average = (student->average + grade) / student->grades_count вычисляет среднее неправильно,
потому что учитывает предыдущий средний балл как сумму, вместо того чтобы складывать все оценки. После
нескольких добавлений среднее будет некорректным.
4. цикл в find_student
Описание ошибки:
Цикл for (int i = 0; i <= manager->capacity; i++) проверяет capacity, а не фактическое кол-во студентов
count. Это может привести к обращению к неиниациализированной памяти и к непонятному поведению программы.
5. цикл вывода оценок в print_student
Цикл for (int i = 0; i <= student->grades_count; i++) идёт на один элемент больше, чем есть в массиве оценок.
Это вызывает выход за пределы массива и может привести к выводу мусора или падению программы.
6. использование realloc без проверки
Вызывается student->grades = (float*)realloc(student->grades, student->grades_count * sizeof(float)); без 
проверки результата. Если realloc вернёт NULL из-за нехватки памяти, старый указатель потеряется, что 
приведёт к утечке памяти и шансы сломанию программы.
